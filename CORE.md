# CORE.md

> **AIへの指示:**
> このファイルは、プロジェクト `nagi` における**最上位の意思決定基準（Constitution）**である。
> コードの実装、UIデザイン、テキストライティングにおいて迷いが生じた場合、常にこのファイルの記述を優先すること。
> ユーザーの利便性や一般的なベストプラクティスよりも、以下の「哲学」が優先される。

## 1. Origin Emotion (原点にある感情)
**「自責からの解放」**
私たちは、スマホを見続けてしまった後の「また時間を無駄にしてしまった」という自己嫌悪を知っている。
このプロダクトは、その自己嫌悪を断ち切るために存在する。
ユーザーを管理・監視するのではなく、「あなたは今、どこにいたのか（石）」と「どこへ行けるのか（波）」を静かに鏡のように映し出すことだけを目的とする。
怒りや罰ではなく、**「許し」と「気づき」**を出発点とする。

## 2. The Transformation (ユーザーの変容)
ユーザーの状態を以下のように変化させることを目指す。

* **Before (Current State):**
    * **拘束 (Stone):** 無意識にデジタル空間に張り付けられ、硬直し、感覚が麻痺している状態。
    * **ノイズ:** 通知や情報の奔流に溺れている。
* **After (Target State):**
    * **解放 (Wave):** デバイスから離れ、現実世界で呼吸し、思考が流動的になっている状態。
    * **静寂 (Nagi):** 自分の時間を取り戻し、凪いだ海のような心持ち。

## 3. Single Truth (唯一の正解)
**「ゼロ・フリクション (Zero Friction)」**
ユーザーに「努力」をさせてはならない。
セットアップ、記録、振り返り、すべてのプロセスにおいて「認知負荷」を極限までゼロにする。
「アプリを開かせること」すら悪とする。
**「ユーザーがこのアプリの存在を忘れている間に、勝手に守ってくれている」**状態こそが、機能的な正解である。

## 4. The Anti-Pattern (絶対にやらないこと)
以下の要素は、たとえリテンション率が上がるとしても**厳格に禁止**する。

* **No Gamification:** バッジ、ストリーク（連続記録）、スコア、ランキングは禁止。人生はゲームではない。
* **No Notification:** アプリ側からユーザーの注意を引くプッシュ通知（リマインダー等）は禁止。
* **No Guilt:** 「スマホを使いすぎです」といった警告や、赤色を使った危険信号の表現は禁止。
* **No Social:** 「シェアする」「いいねする」といった承認欲求を刺激する機能は禁止。

## 5. Texture & Metaphor (手触りとメタファー)
デザインと挙動は、以下のメタファーに従う。

* **The Stone (石):** スマホに拘束されている時間は、重く、硬く、無機質な「石」として表現する。動かないグレーの帯。
* **The Wave (波):** スマホから離れている時間は、柔らかく、常に変化し、有機的な「波」として表現する。パステルカラーの穏やかなアニメーション。
* **Unobtrusive (空気):** UIは主張しない。文字は小さく、余白は大きく。システムフォントやモノクロームを基調とし、装飾的なアイコンは排除する。
